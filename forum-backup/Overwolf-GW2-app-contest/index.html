robotempire.2451:

26 Aug 2014

Some of the API endpoints appear broken as well… https://forum-en.guildwars2.com/external?l=https%3A%2F%2Fapi.guildwars2.com%2Fv1%2Fworld_names.json for example returns a 404. There are others as well.

I’ve tried futzing with the GW2 API before, and it’s really annoying. In order to efficiently do lookups you must create your own database of information. For example, if you want to know the status of Garrison on the green WvW map for the Blackgate-Jade Quarry-Tarnished Coast matchup, that would require multiple API calls each requiring some heavy-handed client-side joining.

My hats off to GW2Spidy which I assume does exactly what I mentioned about building your own database.

I registered for this before I remembered how painful the API is. I feel bad saying that because it’s really awesome an MMO company even made an API available for game data, it’s just not great to work with.

Hey there!

Those are all really good questions which I’ll happily answer.

You as the creator have full rights over your app forever to do with as you wish. We (Overwolf) will distribute your app as a free download to anyone who uses the Overwolf overlay.

Currently, we only offer apps for free. In the future we will implement a system where if the developer of the app wishes they can charge for the app (this is only if the developer wishes, we do not decide this). If the app becomes a paid app, the creator gets percentages of the app sales.

Hopes this answers your questions!
